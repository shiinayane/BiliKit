# BiliKit 测试方案

手动测试清单,覆盖 Float / Theme Sync / Wake Lock 三脚本及其协同。每条给出**步骤**与**理论应观察到的结果**;偏离即为缺陷。

## 0. 准备

- 浏览器:Safari(主目标)+ 任一 Chromium(Chrome/Edge)对照。
- 脚本管理器装齐三脚本:`float` / `theme-sync` / `wake-lock`(均启用)。
- 新安装不含已保存设置时，「视频打开方式」应默认为**新标签页**；执行第 1～11 节抽屉专项前，先在面板手动选「抽屉」。
- 打开开发者工具,留意 Console 报错(理论上**全程无红色报错**)。
- 常用校验片段(控制台):
  - 主题:`document.documentElement.classList.contains('bili_dark')`
  - 抽屉 iframe 数:`document.querySelectorAll('.bk-dframe').length`（首次打开后应始终为 1）
  - 预连接节点:`document.querySelectorAll('link[rel=preconnect]').length`
  - 页面 overflow:`getComputedStyle(document.body).overflow`

## 7. Theme Sync

| # | 步骤 | 预期结果 |
|---|------|----------|
|7.1|系统深色,新开 B 站标签|页面**初始即深色**,无浅→深闪烁(document-start 设 cookie + toggle)|
|7.2|页面打开状态下切换系统主题|当前页**无刷新**秒变;**页面不 reload**(关键回归点)|
|7.3|开多个 B 站标签,切系统主题|**所有标签**一起变,且**都不 reload**|
|7.4|查 cookie:`document.cookie.match(/theme_style=([^;]*)/)?.[1]`|深色为 `dark`、浅色为 `light`|
|7.5|后台搁置一个标签 → 切系统主题 → 切回该标签|该标签恢复可见时主题**已对**(visibilitychange 兜底)|
|7.6|抽屉 iframe 内查是否注入了 theme-sync 逻辑|theme-sync **不在 iframe 运行**(顶层守卫);抽屉主题由 Float 镜像负责|
|7.7|在**播放页**(`/video/*`)tab 切系统主题|也**无刷新**秒变(与首页同机制:swap `bili-theme` 样式表 href + toggle `bili_dark`/`night-mode`)|

## 已知非缺陷(预期内行为)

- **自动播放被拦** → 停在暂停态:浏览器自动播放策略所致,非 bug(3.4)。
- **隐藏顶栏可能留白/错位**:`headerSelectors` 依赖 B 站类名,B 站改版需手动调(3.5)。
- **横滑误关**:理论上 iframe 内长横向列表可能触发;跟手形态下面板会跟随、可反向回弹，误触概率低(2.4)。
- **主题跟系统而非 B 站站内开关**:设计如此(7.x)。
- **重复安装致双开**:旧 `Bilibili-Float` 未删时两份并存;单例守卫兜底，但请手动删旧版(6.7)。
- **抽屉是完整页**:比嵌入播放器重(高清/评论/弹幕换内存);曾试悬停预热整页提速因内存过高已回退。
- **Safari 历史压扁并非全导航压扁**:只处理 B 站跨视频 SPA `pushState`；分 P、真整页跳转、手动/普通标签保持原生语义。
